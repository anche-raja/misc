FORGE is a multi-agent system with dual guardrails designed to modernise enterprise J2EE applications — orchestrating 16 specialised AI agents across a coordinated pipeline that systematically upgrades the full technology stack from Java 8 to Java 21, Spring 5 to Spring 6, and eliminates Struts entirely, replacing every action class, OGNL expression, and struts.xml routing configuration with pure Spring MVC.
The pipeline is anchored by a Leader Agent that coordinates 8 transformation agents running on Claude Sonnet 4.5 — each a specialist in one migration domain — while 5 independent review agents powered by Amazon Nova Pro, a completely different model family, validate every transformation before it can proceed; the dual guardrails layer enforces enterprise security standards and naming conventions at both ends of the pipeline, ensuring nothing enters the transformation and nothing leaves the quality gate without passing explicit safety checks.
Grounded by a Bedrock RAG Knowledge Base containing your organisation's coding standards, architecture decisions, and Spring migration patterns, FORGE delivers a Struts-free, OGNL-free, cloud-ready codebase on AWS — transforming 18 months of high-risk manual J2EE migration into a 5-month, auditable, AI-orchestrated process where every file carries a complete decision trail: the model pair used, the review score, the guardrail verdicts, and the full retry history.
